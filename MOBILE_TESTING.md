# Как протестировать мобильную версию на десктопе

## Способ 1: Инструменты разработчика браузера (DevTools)

### Chrome / Edge / Brave:

1. **Откройте приложение** в браузере (например, `http://localhost:8000`)

2. **Откройте DevTools** одним из способов:
   - Нажмите `F12`
   - Или `Ctrl+Shift+I` (Windows/Linux) / `Cmd+Option+I` (Mac)
   - Или правый клик → "Проверить элемент" / "Inspect"

3. **Включите режим устройства**:
   - Нажмите кнопку **Toggle device toolbar** (иконка телефона/планшета) в верхней панели DevTools
   - Или нажмите `Ctrl+Shift+M` (Windows/Linux) / `Cmd+Shift+M` (Mac)

4. **Выберите устройство**:
   - В верхней панели выберите устройство из списка (iPhone, iPad, Samsung Galaxy и т.д.)
   - Или введите свои размеры вручную

5. **Проверьте разные устройства**:
   - iPhone SE (375x667)
   - iPhone 12/13/14 (390x844)
   - iPhone 14 Pro Max (430x932)
   - Samsung Galaxy S20/S21 (360x800)
   - iPad (768x1024)
   - iPad Pro (1024x1366)

### Firefox:

1. Откройте приложение в браузере
2. Нажмите `F12` для открытия DevTools
3. Нажмите кнопку **Responsive Design Mode** (иконка телефона/планшета)
   - Или `Ctrl+Shift+M` (Windows/Linux) / `Cmd+Shift+M` (Mac)
4. Выберите устройство из списка или введите размеры вручную

### Safari (Mac):

1. Включите меню "Разработка" в настройках Safari:
   - Safari → Настройки → Дополнения → Поставьте галочку "Показывать меню разработки"
2. Откройте приложение в Safari
3. Разработка → Войти в адаптивный режим
   - Или `Cmd+Option+R`

## Способ 2: Локальный сервер на мобильном устройстве

Если у вас уже запущен локальный сервер (например, `python3 -m http.server 8000`):

1. **Узнайте IP-адрес вашего компьютера**:
   - Windows: `ipconfig` в командной строке, найдите IPv4 адрес
   - Mac/Linux: `ifconfig` или `ip addr`, найдите адрес (обычно начинается с 192.168.x.x)

2. **Убедитесь, что мобильное устройство в той же Wi-Fi сети**, что и компьютер

3. **Откройте на мобильном устройстве**:
   ```
   http://ВАШ_IP:8000
   ```
   Например: `http://192.168.1.100:8000`

4. **Если не открывается**, проверьте:
   - Файрволл не блокирует порт 8000
   - Оба устройства в одной сети Wi-Fi
   - IP-адрес правильный

## Способ 3: Использование эмуляторов

### Android Studio Emulator:
- Установите Android Studio
- Создайте виртуальное устройство (AVD)
- Запустите эмулятор
- Откройте браузер в эмуляторе и введите `http://10.0.2.2:8000` (это специальный адрес для доступа к localhost хоста)

### iOS Simulator (только Mac):
- Установите Xcode
- Запустите iOS Simulator
- Откройте Safari в симуляторе
- Введите `http://localhost:8000` или IP-адрес вашего Mac

## Рекомендации для тестирования:

1. **Проверьте разные размеры экрана**:
   - Маленькие телефоны (320-375px ширина)
   - Средние телефоны (375-414px)
   - Большие телефоны (414-430px)
   - Планшеты (768px+)

2. **Проверьте ориентацию**:
   - Портретная (вертикальная)
   - Альбомная (горизонтальная)

3. **Проверьте touch-события**:
   - Тапы по кнопкам
   - Свайпы (если есть)
   - Масштабирование

4. **Проверьте производительность**:
   - Скорость загрузки
   - Плавность анимаций
   - Отзывчивость интерфейса

## Быстрый способ (рекомендуется):

**Используйте Chrome DevTools** - это самый быстрый и удобный способ:
1. Откройте `http://localhost:8000`
2. Нажмите `F12`
3. Нажмите `Ctrl+Shift+M` (или `Cmd+Shift+M` на Mac)
4. Выберите устройство из списка
5. Готово! Теперь вы видите мобильную версию
